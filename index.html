3. Types of Joins in Tableau:

Once both tables are in the Data Source tab, Tableau automatically suggests an inner join,

but you can modify the type of join depending on the scenario.


a. Inner Join:


 Description: Returns only records where there is a match in both tables.


 How to Create in Tableau:


 Drag the first table to the canvas (Demographics).

 Select Open from the menu or double-click the first table to open the join

canvas (physical layer).

 Double-click or drag another table to the join canvas (Salary)

 Click the join icon to configure the join. Add one or more join clauses by

selecting a field from one of the available tables used in the data source,

choosing a join operator, and a field from the added table..

 Tableau automatically detects the common field (EmployeeID). If not,

manually select it.

 Choose Inner Join in the Join Type options.

 Result: You will see only employees whose employee id matches in

both Demographics and Salary table.

b. Left Join:


 Description: Returns all records from the left table (Demographics), and matched

records from the right table (salary). If there’s no match, NULL values are returned for

fields from the right table.

 How to Create in Tableau:

 In the join settings, select Left Join.

 Result: All employees will be returned, even if data missing in Salary. Salary information

will be NULL for those without a match.

Introducing Power BI –Components and the flow of work. Power BI Desktop Interface-

Introduction to Power BI

Objective

To introduce Power BI, its main components, and the workflow, and to understand the Power

BI Desktop interface, including the five main areas of the Report view.


Prerequisites

 Microsoft Power BI Desktop installed.


Theory

Power BI is a business intelligence and data visualization tool by Microsoft that enables users

to analyze data and share insights. It consists of various components, including Power BI

Desktop, Power BI Service, Power BI Mobile, and Power BI Report Builder.

Components of Power BI

1. Power BI Desktop: The main application for designing and developing reports, with

tools for data preparation, modeling, and visualization.

2. Power BI Service: A cloud-based platform that allows users to share, publish, and

access reports and dashboards online.

3. Power BI Mobile: An app for viewing and interacting with reports and dashboards on

mobile devices.

4. Power BI Report Builder: A tool for creating paginated reports suitable for printing,

often used for detailed, document-style reporting.

`



Flow of Work in Power BI

1. Data Import: Power BI connects to various data sources, such as Excel, SQL Server,

and web data, allowing users to import and combine data.

2. Data Transformation: Using Power Query, data is cleaned, transformed, and shaped

to prepare it for analysis.

3. Data Modeling: Relationships are created between tables, calculations are added, and

a data model is built to support analysis.

4. Data Visualization: Interactive charts, graphs, and maps are created in Power BI

Desktop to present insights effectively.

5. Publishing: Reports are published to the Power BI Service, where they can be shared

and viewed by others.

6. Access and Collaboration: Reports and dashboards can be accessed online via the

Power BI Service or on mobile devices, and users can collaborate and make data-driven

decisions.

Power BI Desktop Interface

The Power BI Desktop application provides an interface for developing and formatting

reports. The Report View in Power BI Desktop has five main areas:

1. Ribbon: Located at the top, the Ribbon includes menus for Home, Insert, Modeling,

View, and Help. It provides access to data import, transformation, visualization options,

formatting, and publishing tools.

2. Report View Canvas: The central area where visuals (charts, tables, and graphs) are

created and arranged. The canvas serves as a workspace for designing the layout of a

report.

3. Fields Pane: Located on the right, this pane lists all tables and fields in the imported

dataset. Users drag fields from this pane to add data to visualizations.

4. Visualizations Pane: Also on the right, this pane contains various chart types,

formatting options, and analytical tools. Users select a visualization type and customize

it by adding data and formatting features.

5. Pages Tab: Found at the bottom, this area shows tabs for each report page, allowing

users to create multi-page reports for better organization and detail.

Steps for Creating a Basic Report in Power BI Desktop

1. Import Data: Go to Home > Get Data, select a data source, and load it into Power BI.

2. Transform Data: Use Transform Data in the Ribbon to open Power Query, where

you can clean and shape data as needed.

3. Create Visuals:

o Go to the Visualizations Pane and select a chart type.

o Drag fields from the Fields Pane to populate the chart.

4. Add Report Pages: Use the Pages Tab to add additional report pages and organize

visuals.

5. Save and Publish: Save your report and publish it to the Power BI Service if needed
